Finished, running at home

Mojiokoshi Transcribe + summary-copy helper

Drop a video or audio file in from the browser and local Whisper (small, CPU) transcribes it, showing progress in real time. When it is done, pick from three presets — lecture notes, key bullet points, or exam prep — and one-tap copy the prompt plus the full text to paste into ChatGPT. It includes history and txt/srt downloads. The server is my home PC; no external API is used.

Where I got stuck, and how I got out

01

On an HTTP page over the home LAN, navigator.clipboard is not available, so the crucial copy fails.

I added an execCommand fallback, and if that also fails, it shows a pre-selected textarea, so a way to copy always remains.

02

Transcribing on CPU takes a long time, yet by default there is no way to see progress.

I estimate progress from the total length via ffprobe and the timestamps of the segments Whisper emits one by one, and stream the text being recognized in real time too.

03

I was not sure how to hand a long full transcript over to ChatGPT.

The default is a one-tap copy of the preset prompt plus the full text, with a second option to copy in chunks only when the text is long.
