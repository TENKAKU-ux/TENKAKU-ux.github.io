# サイト英語版のフォーマット（毎回の更新で従う正本）

このサイトは日本語（`/`）と英語（`/en/`）の2言語を1つのデータから生成する。
**日本語を足したら、同じ更新の中で必ず英語も揃える。** 揃え方はこのファイルの手順どおり。

英語は「別ページを手書きする」のではなく、日本語データに **`en` オーバーレイ** を足すだけ。
`scripts/build.mjs` の `localize()` が、日本語の各項目に `en` の同じ形を重ねて英語ページを作る。
配列はインデックスで対応づける（`en` 側は翻訳する要素だけ書けばよく、`name`/`url`/`src` などは日本語側から引き継がれる）。
`en` に書かなかったフィールドは日本語のまま出る（部分的でも壊れない）ので、**訳し忘れ＝日本語混じり**になる。下のチェックで防ぐ。

## 1. `data/projects.json` — 作品を1つ足す/直すたび

**区分（必須）**: 各作品に `"category"` を付ける。作品一覧（`/projects/`）はこれで軽く仕切られる。
- `"published"` … 公開しているもの（誰でも試せる・ソース公開）
- `"local"` … このパソコンで使っているもの（未公開・自分の環境で運用）
未知の値／未設定でも消えはしないが、区分の見出し無しで末尾にまとまるので必ず設定する。見出し・説明文は UI 側（build.mjs の `STR…projectGroups`・日英対）にあり、区分キーだけデータに置く。

日本語ブロックの末尾（`tags` の後）に `en` ブロックを足す。訳すのは日本語の文章が入るフィールドだけ:

```jsonc
"en": {
  "title":       "…",   // 例: 速・打 → Soku-Da（固有名はローマ字 or そのまま）
  "subtitle":    "…",
  "status":      "…",   // 例: 公開中 → Live
  "summary":     "…",
  "description": "…",
  "period":      "…",   // 日付だけなら省略可（日本語＝英語）。括弧の注記があるときは訳す
  "costApprox":  "…",   // ★下の費用ルールを必ず反映
  "aiUsed":      [ { "usage": "…" }, … ],            // name は引き継ぐので usage だけ
  "stuckPoints": [ { "problem": "…", "solution": "…" }, … ],
  "screenshots": [ { "alt": "…" }, … ],              // src は引き継ぐので alt だけ
  "links":       [ { "label": "…" }, … ],            // url は引き継ぐので label だけ。英語で十分なら省略可
  "article":     { "label": "…" }                    // Zenn 記事リンク（あれば）。url は引き継ぐので label だけ
}
```

- **順番はインデックスで対応する。** `aiUsed`/`stuckPoints`/`screenshots`/`links` は日本語と同じ件数・同じ並びにする。
- 日本語側と英語で意味が変わらない値（`tags`・`tools`・技術名・URL）は書かない＝自動で共通。
- `related`（他の方々の作品）の `title`/`author` は固有名なので原則そのまま。見出しや「作者:」ラベルは UI 側（build.mjs）で自動的に英語になる。
- `article`（Zenn 記事リンク・作品にあれば）は `en` 側で `label` だけ訳す（`url` は共通）。

## 2. `data/links.json` — リンクを足す/直すたび

各リンクに `en` を足す。`title` が既に英語なら `note` だけでよい:

```jsonc
{ "title": "…", "url": "…", "note": "日本語の説明。", "en": { "note": "English description." } }
```

## 3. `scripts/build.mjs` の `STR` — 画面の「枠」を変えたとき

見出し・ナビ・定型文（「実録データ」「使った AI」「あゆみ」やヒーローの文など）は `STR.ja` / `STR.en` に対で入っている。
**枠の文言を変える／`history` に1行足すときは、必ず `ja` と `en` の両方を直す。**（作品・リンクの中身はここではなく `data/*.json`。）

## 4. 費用欄ルール（日英どちらも厳守）

`costApprox` は、どの作品でも「大前提として Claude / Codex のサブスク（各 月約3,000円）を常用している」ことが分かるように書く。個別費用に換算しない。
英語表現の定番: `Baseline: Claude and Codex subscriptions (about ¥3,000/month each) are always running as a given.`

## 5. ビルドとチェック

```bash
node scripts/build.mjs
```

- `/` と `/en/` の同じページを見比べ、**英語ページに日本語が残っていないか**を確認（残っていたら `en` の訳し忘れ）。日本語ページはこれまでどおり。
- 言語切替（ヘッダ右の EN / 日本語）が、見ているページの相手言語版に飛ぶか。
- 画像・外部リンクは日英共通なので変わらない。

英語版を足したくない小さな更新でも、`en` を書かなければ英語ページはその項目だけ日本語で出る。
公開する以上は基本そろえる方針。訳を保留するなら、その旨をユーザーに一言伝えること。
